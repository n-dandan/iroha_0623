### 6-1. メインカラー（7択ボタン・単一選択）

```js
const MAIN_COLORS = [
  { key: "red",        label: "赤",       hint: "#d8453c" },
  { key: "orange",     label: "オレンジ", hint: "#e8843c" },
  { key: "yellow",     label: "黄",       hint: "#e8c23c" },
  { key: "green",      label: "緑",       hint: "#4ca85f" },
  { key: "blue",       label: "青",       hint: "#3c6fe8" },
  { key: "purple",     label: "紫",       hint: "#8a5ce8" },
  { key: "monotone",   label: "モノトーン", hint: "#5a6271" },
];
```

- 単一選択。もう一度押すと解除（おまかせに戻る）。
- 未選択（おまかせ）でも生成可能。
- `hint` はボタン上の小さな色見本に使う（実際の生成色ではない）。

### 6-2. トンマナ（4軸スライダー）

**見た目はスライダー、中身は3値（左／中央／右）。クリックで3ポジションにスナップ。** ドラッグ不要、トラックの左/中/右をクリックでつまみが吸着する。

```js
const TONE_AXES = [
  { key: "form",    left: "やわらか",   right: "かっちり" },
  { key: "era",     left: "クラシック", right: "モダン" },
  { key: "energy",  left: "にぎやか",   right: "静か" },
  { key: "class",   left: "カジュアル", right: "ラグジュアリー" },
];
```

- 各軸の値：`-1`（左）/ `0`（中央）/ `1`（右）。
- 初期値：全軸 `0`（中央＝おまかせ）。
- **中央（0）の軸はプロンプトに含めない**（指定なし扱い）。
- 両端を選んだ軸だけ「{left}寄り」「{right}寄り」とプロンプトに加える。

### 6-3. 用途トグル（デジタル/印刷）

```js
// "digital" | "print"
```

- `"digital"`：プレビューは Web風。コピーは HEX/RGB/CSS変数。
- `"print"`：プレビューは チラシ風。コピーに **CMYK参考値** を追加。生成プロンプトに彩度抑制制約を追加。

### 6-4. フォント分け（ラジオ）

```js
// "split" | "single"
```

- `"split"`：タイトルと本文で別フォントを提案。
- `"single"`：**タイトル＝本文で1フォント**。

### 6-5. 日英の組み合わせ（ラジオ）

```js
// "bilingual" | "jp_only"
```

- `"bilingual"`：日本語フォント＋英語フォントの組み合わせで提案。
- `"jp_only"`：**日本語フォントのみ**で提案。

### 6-6. 自由記入

```
textarea / placeholder: 例：老舗の和菓子屋のチラシを作りたい
```

> **優先順位ルール**：メインカラー/トンマナの選択と自由記入が矛盾する場合は、**自由記入を優先**する（システムプロンプトで明示）。

## 7. Gemini API 呼び出し（callGemini）

```js
// app.js 先頭
const { GEMINI_API_KEY, MODE, MODEL, PROXY_URL } = window.IROHA_CONFIG;
const CONFIG = { MODE, MODEL, PROXY_URL };

// ★ 本番化のときはこの関数の中身だけ差し替える
async function callGemini(bodyPayload) {
  if (CONFIG.MODE === "proxy") {
    const res = await fetch(CONFIG.PROXY_URL, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(bodyPayload),
    });
    if (!res.ok) throw new Error(`プロキシエラー HTTP ${res.status}`);
    return await res.json();
  } else {
    const url = `https://generativelanguage.googleapis.com/v1beta/models/${CONFIG.MODEL}:generateContent?key=${encodeURIComponent(GEMINI_API_KEY)}`;
    const res = await fetch(url, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(bodyPayload),
    });
    if (!res.ok) {
      const e = await res.json().catch(() => ({}));
      throw new Error(e?.error?.message || `APIエラー HTTP ${res.status}`);
    }
    return await res.json();
  }
}
```

> **本番化のとき**：`config.js` の `MODE` を `"proxy"` に変更し、`GEMINI_API_KEY` をサーバ側（PHP の `.env` や環境変数）に移す。`app.js` 自体は変更不要。

### 7-1. プロンプト組み立て（buildPrompt）

```js
function buildPrompt({ mainColor, tone, purpose, fontMode, langMode, free }) {
  const parts = [];

  // メインカラー（おまかせ時は含めない）
  if (mainColor) parts.push(`メインカラーの方向性：${mainColorLabel(mainColor)}`);

  // トンマナ（中央=0 の軸は含めない）
  const toneParts = [];
  TONE_AXES.forEach(ax => {
    const v = tone[ax.key];
    if (v === -1) toneParts.push(`${ax.left}寄り`);
    if (v ===  1) toneParts.push(`${ax.right}寄り`);
  });
  if (toneParts.length) parts.push(`トンマナ：${toneParts.join("、")}`);

  // 用途
  parts.push(purpose === "print" ? "用途：印刷物" : "用途：デジタル（Web）");

  // フォント分け
  parts.push(fontMode === "single"
    ? "フォント：タイトルと本文で同一のフォントを使う"
    : "フォント：タイトルと本文で別のフォントを使う");

  // 日英
  parts.push(langMode === "jp_only"
    ? "フォント言語：日本語フォントのみ"
    : "フォント言語：日本語フォントと英語フォントの組み合わせ");

  // 自由記入（最優先）
  if (free?.trim()) parts.push(`自由記入（最優先で反映）：${free.trim()}`);

  if (!parts.length) parts.push("おまかせで、汎用的に使いやすい上質な配色とフォントを提案してください。");
  return parts.join("\n");
}
```

### 7-2. システムプロンプト（sys 配列 → join）

```
あなたは日本語UI向けのアートディレクターです。指示に合う「デザインの方向性」を必ず2案提案します。
2案は互いにムード・配色・フォントの方向性に明確な差をつけ、ユーザーが選ぶ意味があるようにします。
各案は配色5色とフォントペアのセットで構成します。

【配色】
各案の配色は必ず5色で、role を primary / secondary / accent / base / text に1つずつ割り当てます。
base は背景向けの明るい色、text は base 上で十分読める文字色にします（コントラスト比4.5以上を意識）。
name は各色の短い日本語名（例：朱赤、墨、生成り）。

【フォント】
フォントは必ず次のカタログの family からのみ選びます（存在しない名前を作らない）。
日本語の本文・見出しには subset が jp のフォントを選びます。Latin 専用フォントを日本語本文に使わないこと。
カタログ: {fontCatalog の JSON}

【指示の優先順位】
ユーザーの「自由記入」が他の選択（メインカラー・トンマナ）と矛盾する場合は、自由記入を優先します。

【印刷用途のとき】
用途が「印刷物」の場合、印刷で再現が難しい高彩度・蛍光色を避け、彩度をやや抑えた配色にします。

【各案に含めるテキスト】
theme  … 提案の方向性を表す短い日本語のテーマ名（15文字以内）
reason … なぜこの配色・フォントがその指示に合うかの理由（1〜2文）
fontReason … フォントペア全体の選定理由（1文。タイトルと本文をまとめて1文）
```

> **トークン節約**：`fontCatalog` は FONT_LIST から `family` / `subset` / `category` の3フィールドだけ抜き出す。`weights` はプロンプトに含めない。

## 8. 検証ガード（Gemini 出力の安全化）

各案（proposal）ごとに以下を適用する。

### 8-1. guardPalette（パレット検証）

1. 5色すべてに role（primary/secondary/accent/base/text）が揃っているか確認。
2. 足りない role はフォールバック値 `#888888` で補完。
3. **コントラスト比チェック**：`base` と `text` の WCAG コントラスト比が 4.5 未満なら、`text` の明度を補正（base が明るければ text を暗く `L=18%`、暗ければ明るく `L=92%`）。

### 8-2. resolveFont（フォント検証）

1. Gemini が返した `family` が `FONT_LIST` に存在するか確認（大文字小文字を無視）。
2. 存在しない → フォールバック：日本語が必要なら `subset="jp"` の先頭、それ以外はリスト先頭。
3. `weight` が該当フォントの `weights` に存在するか確認。
4. 存在しない → **最近傍のウェイトに丸める**（例：800 → 700）。
5. `fontMode === "single"` のときは body を heading と同一にそろえる。

## 14. 色ユーティリティ（app.js）

```js
const C = {
  hexToRGB(hex)       // → { r, g, b }
  hexToHSL(hex)       // → { h, s, l }
  hslToHex(h, s, l)   // → "#rrggbb"
  rgbToCMYK(r, g, b)  // → { c, m, y, k }  ※単純変換・参考値
  lum(hex)            // 相対輝度
  contrast(a, b)      // コントラスト比
  textOn(bg)          // 輝度 > 0.4 なら '#20242f'、それ以外 '#ffffff'
  valid(h)            // /^#[0-9A-Fa-f]{6}$/ テスト
}
```

### rgbToCMYK（参考値・単純変換）

```js
rgbToCMYK(r, g, b) {
  const r1 = r / 255, g1 = g / 255, b1 = b / 255;
  const k = 1 - Math.max(r1, g1, b1);
  if (k === 1) return { c: 0, m: 0, y: 0, k: 100 };
  const c = (1 - r1 - k) / (1 - k);
  const m = (1 - g1 - k) / (1 - k);
  const y = (1 - b1 - k) / (1 - k);
  return {
    c: Math.round(c * 100),
    m: Math.round(m * 100),
    y: Math.round(y * 100),
    k: Math.round(k * 100),
  };
}
```

> ICC プロファイル非対応の単純式。Adobe 製品の値とは一致しない。**必ず「参考値」と表示する。**
